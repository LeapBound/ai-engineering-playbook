# AI 工程治理方法论

## 为什么（风险）
AI 工程引入后，代码生成速度提升，但工程决策、规则一致性和质量门禁容易被稀释，导致工程失控风险上升。

## 不做会怎样
缺乏治理方法论会让团队在高速度迭代中失去架构一致性与质量可控性，最终出现难以维护的系统与不可预测的交付风险。
